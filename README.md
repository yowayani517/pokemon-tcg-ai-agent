# Pokémon TCG AI Battle — 対戦エージェント開発

Kaggle「The Pokémon Company - PTCG AI Battle Challenge」向けの、ポケモンカードゲームを自動対戦するAIエージェントの開発リポジトリ。
**ルールベース → 強化学習 → プランニング → 前方探索** と複数アプローチを実装・比較し、現メタに対して最も強い手法を実証的に選定した記録。

> 本リポジトリには **自分が書いたコードと検証結果のみ** を載せています。競技中のため、他プレイヤーのデッキ・リプレイ・コンペ公式SDKは含みません（`.gitignore` で除外）。

| アプローチ | 対メタ勝率 | サイズ | 特徴 |
|---|---|---|---|
| ルールベース壁 | 82% | 小 | 受け特化。操縦改善は頭打ち |

## アプローチの変遷と学び

### 1. ルールベース（`agent/main.py`）
合法手を優先度ルールでスコアリングして選ぶベースライン。エネルギー配分（攻撃要員に集中→余りはベンチ）や、犠牲・引きずり出しなどの臨機応変な立ち回りを実装。
**学び**: 受け身のデッキは「判断の数」が少なく、操縦をいくら磨いても勝率は頭打ちになる。

### 2. 自己対戦モンテカルロ強化学習（`experiments/self_play_rl.py`, `build_rl_agent.py`）
各決定を試合の勝敗でラベル付けし、`f(盤面, 選択肢) → 勝率` を LightGBM で学習。ε-greedyで探索しながら方策反復。
学習済みモデルを **numpy配列にエクスポートして依存ゼロの推論コードに変換**（提出環境に外部ライブラリを持ち込めない制約への対応）。
**学び**: 判断の多いアグロ/コンボデッキでは有効。ただし不確定性とファイルサイズ制約がネック。

### 3. プランニング（`agent/main_plan.py`）⭐ 採用
**「ターン開始時に攻撃プランを1つ確定し、その後の全ての手をそのプランに従属させてスコアリング」** する設計。
1手ごとに独立評価する方式と違い、「進化 → エネ充填 → 引きずり出し → 攻撃」というコンボを**逆算して一筆書きで組み立てる**。

主な実装:
- ターンの攻撃ゴール（誰の・どの技で・どの相手を倒すか）を全探索で確定し、付与/進化/入替/サポートをゴールに合わせて選択
- マッチアップ別カウンター（特定アーキタイプ検知 → 立ち回り変更）
- タイプ相性（弱点×2／抵抗−30）の計算、状態異常（こんらん）回避、リソース管理
- 全体 try-except によるクラッシュセーフ設計

### 4. 前方探索（`agent/main_search.py`）
コンペSDKが提供する **本物の前方シミュレータ（search API）** を使い、各候補手を自ターン終了までロールアウトして盤面評価する版。
**結果は正直に「負け」**: 0.3〜0.7秒/試合と高速で動作するが、ポケカは山札シャッフル・相手手札非公開の不確定ゲームのため、予測山1本のロールアウトに基づく初手選択がノイズとなり、よく調整されたプランニングを下回った（対メタ53%）。
本気で勝たせるにはモンテカルロ多数ロールアウト＋相手デッキ予測が必要。**将来の発展用に土台として保管**。

## 検証基盤

- `experiments/gauntlet_*_iso.py` + `run_one_*.py`: **1試合=1サブプロセスの隔離ガントレット**。
  - 理由①: 対戦エンジン(dll)は同一プロセスで多数試合を回すと状態が壊れるため、プロセス隔離で正確に測定。
  - 理由②: 評価は N=10 だと ±15% ブレるため、本命の比較は **N=30** で実施しノイズと本物の差を分離。
- `experiments/analyze_replays.py`, `narrate_replay.py`: リプレイ解析・プレイの言語化によるメタ分析。
- `experiments/verify_submission.py`: 提出物が例外を出さないかの事前検証。
